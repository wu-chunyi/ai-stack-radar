# TF-IDF / 余弦相似度 / 检索引擎 / Embedding 详解

> 对应代码：`src/tools/rag.ts`，137 行，5 个函数 + 1 个类。

---

## 全流程总览

```
文档 → 切块 → 向量化 → 存起来
                ↑
          TF-IDF（数词频）或 Embedding（调 API）

用户提问 → 向量化 → 跟每个 chunk 算余弦相似度 → 取 top-3 → 塞进 prompt → LLM 回答
```

---

## 1. 文档切块（chunkMarkdown，第 29–53 行）

**做什么**：按 markdown 标题（#/##/###）把文档切成段落大小的块。

**为什么**：LLM context 有上限，不能把 440 行全塞进 prompt。先切小，再只找相关的。

**`length > 30` 过滤**：太短的块（只有分割线 `---`）没检索价值，直接丢弃。

---

## 2. TF-IDF（第 55–86 行）

### TF（词频）= 这个词在这个 chunk 里有多常见

```
公式：TF(词, chunk) = 该词在 chunk 中出现次数 / chunk 总词数

例：chunk 有 10 个词，"v0" 出现 2 次 → TF = 2/10 = 0.2
```

**问题**："的"、"是" 在每个 chunk 里都高频，但毫无检索价值。

### IDF（逆文档频率）= 这个词有多罕见

```
公式：IDF(词) = log(总 chunk 数 / 包含该词的 chunk 数)

例：28 个 chunk，"chromatic" 只出现在 3 个 → IDF = log(28/3) = 2.23（罕见，有区分力）
例：28 个 chunk，"的" 出现在 28 个       → IDF = log(28/28) = 0（到处都有，没用）
```

### TF × IDF = 最终权重

```
"chromatic" 在 chunk A：0.2 × 2.23 = 0.446（高权重 = 特征词）
"的"       在 chunk A：0.3 × 0     = 0    （零权重 = 废词）
```

**一句话**：一个词在这个 chunk 里多（TF 高）且在别的 chunk 里少（IDF 高），才是特征词。

---

## 3. 余弦相似度（第 88–98 行）

每个 chunk 变成一个向量后，用**两个向量之间的夹角**衡量相似度：

```
公式：cos(θ) = (A·B) / (|A| × |B|)

同向（完全相关）→ cos = 1
垂直（完全无关）→ cos = 0
```

**为什么用余弦不用欧氏距离**：余弦只看方向不看长度。一篇长文档和一个短问句，词频绝对值差很远，但"关于同一个话题"时方向一致 → 余弦高。

---

## 4. 检索引擎（TfIdfRetriever，第 100–136 行）

### index()：离线建索引（一次性）

```
切块 → 分词 → 算全局 IDF → 算每个 chunk 的 TF×IDF 向量 → 存起来
```

### search()：在线检索（每次提问）

```
query 也变成 TF-IDF 向量 → 跟每个 chunk 算余弦 → 按分数排序 → 取 top-3
```

---

## 5. Embedding 方式有什么不同

### 整个流程只换了一步

```
TF-IDF：文档 → 切块 → 分词 → 数词频×IDF → 稀疏向量 → 余弦
Embedding：文档 → 切块 → 调 API        → 稠密向量 → 余弦
                         ↑↑↑ 只换这里 ↑↑↑
```

切块、余弦比较、top-K、塞 prompt 全部不变。

### 向量长什么样

```
TF-IDF（稀疏，几千维，大部分是 0）：
  { "chromatic": 0.446, "storybook": 0.38, 其他几千词: 0 }

Embedding（稠密，固定 1536 维，每维都有值）：
  [0.012, -0.034, 0.089, ..., -0.015]
```

### 为什么 Embedding 能搜到 TF-IDF 搜不到的

```
TF-IDF：
  query "Phase 4 issue"  词集 = {phase, issue}
  chunk "#27 写 E2E"     词集 = {27, 写, e2e}
  交集 = 空 → 余弦 = 0 ❌

Embedding：
  query → [0.12, -0.03, 0.45, ...]   ← "项目规划"语义区
  chunk → [0.10, -0.05, 0.42, ...]   ← 也在"项目规划"语义区
  两向量接近 → 余弦 ≈ 0.85 ✅
```

Embedding 模型训练时学会了"issue"和"#27 E2E 测试"语义相近。纯数学的 TF-IDF 做不到。

## 面试答题模板

> "TF-IDF 和 Embedding 的区别？"

| | TF-IDF | Embedding |
|---|---|---|
| 原理 | 数词频 × 逆文档频率 | 神经网络编码语义 |
| 向量 | 稀疏，维度 = 词表大小 | 稠密，固定维度（1536） |
| 同义词 | ❌ 搜不到 | ✅ 能搜到 |
| 成本 | 零（纯数学） | 调 API 或跑模型 |
| 速度 | 极快 | 慢（网络延迟） |
| 适用 | 关键词明确的场景 | 自然语言问答 |
| 生产实践 | 常与 Embedding 混合使用（Hybrid Search） | 主流 RAG 标配 |
