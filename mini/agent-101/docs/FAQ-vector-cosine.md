# 向量与余弦相似度 · 为什么选这两个工具

---

## 向量是什么

一列有顺序的数字，代表空间里的一个点。

```
1 维: 3.5               → 数轴上的点
2 维: [3.5, 2.0]        → 平面上的点
1024维: [0.12, -0.03, ...] → 1024维空间里的点（Embedding 输出）
```

在 RAG 里：每个 chunk 和每个 query 都是 1024 维空间里的点。
语义相似 = 两个点方向相近。

---

## 为什么不用欧氏距离

欧氏距离 = 两点间直线距离，受文本长度影响：

```
"我喜欢狗"          → 向量长度小 [0.5, 0.3, ...]
"我非常非常非常喜欢狗" → 向量长度大 [2.0, 1.2, ...]

欧氏距离 = 很大 → 判定"不相似" ❌（但语义明明一样！）
```

---

## 余弦相似度：只看方向，不看长度

把向量看成从原点出发的箭头，余弦相似度 = 两个箭头夹角的余弦值。

```
夹角 0°  → cos = 1.0  → 完全相似
夹角 90° → cos = 0.0  → 不相关
夹角 180°→ cos = -1.0 → 完全相反
```

"我喜欢狗" 和 "我非常非常喜欢狗"：
  方向几乎相同 → 夹角 ≈ 0° → 余弦 ≈ 1.0 ✅

---

## 公式推导（3 维例子）

```
A = [1, 2, 3]   B = [4, 5, 6]

第1步 点积: dot = 1×4 + 2×5 + 3×6 = 32
第2步 长度: |A| = √(1+4+9) = 3.74   |B| = √(16+25+36) = 8.77
第3步 结果: cos = 32 / (3.74×8.77) = 0.975  → 非常相似
```

---

## 代码与公式对照

```ts
// cosineDense(a, b) in dashscope-embed.ts
let dot = 0, magA = 0, magB = 0;
for (let i = 0; i < a.length; i++) {
  dot  += a[i]! * b[i]!;    // 点积：对应维度相乘再累加
  magA += a[i]! * a[i]!;    // |A|²
  magB += b[i]! * b[i]!;    // |B|²
}
return dot / (Math.sqrt(magA) * Math.sqrt(magB));  // cos = 点积/(|A|×|B|)
```

## 四种方法对比

| 方法 | 公式 | 是否受文本长度影响 | 用在 RAG |
|---|---|---|---|
| 欧氏距离 | √Σ(aᵢ-bᵢ)² | ✅ 受影响 | ❌ 不用 |
| 点积 | Σ(aᵢ×bᵢ) | ✅ 受影响（长向量分高） | 有时用（归一化后=余弦） |
| 曼哈顿距离 | Σ|aᵢ-bᵢ| | ✅ 受影响 | ❌ 不用 |
| **余弦相似度** | 点积/(|A|×|B|) | **❌ 不受影响** | **✅ 标准选择** |

## 面试答题模板

> "为什么用余弦相似度而不是欧氏距离？"

```
文本的语义相似度应该只取决于「方向」，不取决于「长度」。
"我喜欢狗" 和 "我非常非常喜欢狗" 语义相同，
但后者向量的绝对值更大，欧氏距离会认为它们不相似。
余弦通过除以两个向量的长度做了归一化，
使比较只关注方向（语义），而非大小（文本长度）。
```
