# Embedding 工作原理 · 维度 · 理论基础

---

## Embedding 做了几件事

**一件事，用同一个模型做两次：**

```
离线（建索引）：chunk 文本 → 模型 → 向量 → 存到向量数据库
在线（搜索时）：用户 query → 同一个模型 → 向量 → 跟存好的算余弦
```

没有别的。区别只是一个是批量、一个是实时。

---

## 1024 维代表什么

每一维没有人类可读的含义，1024 维共同编码了文本的「语义坐标」。

```
TF-IDF：第 0 维 = "playwright" 权重（你能看懂每一维）
Embedding：第 0 维 = 0.12（你看不懂，但 1024 个数加在一起 = 语义位置）
```

类比：GPS 坐标用经纬度 2 个数描述地球上的位置；Embedding 用 1024 个数描述「语义空间」里的位置。

### 维度越高越好吗？不是，收益递减

| 维度 | MTEB 效果 | 存储/chunk | 适用场景 |
|---|---|---|---|
| 256 | ~60% | 1 KB | 移动端 |
| 512 | ~62% | 2 KB | 成本敏感 |
| **1024** | **~64.5%** | **4 KB** | **主流甜蜜点** |
| 3072 | ~65.2% | 12 KB | OpenAI 旗舰 |

256→1024：效果 +4.5%（值得）
1024→3072：效果 +0.7%，存储 ×3（性价比差）

**1024 是行业甜蜜点**。BGE/GTE/阿里百炼都选 1024。

## 理论基础：为什么高维向量能编码语义

**三个理论，一脉相承：**

### 1. 分布假说（1957）

Firth："You shall know a word by the company it keeps"
（一个词的含义由它的上下文决定）

```
"我遛了我的___"  → 狗、猫、仓鼠（不是汽车）
"我停了我的___"  → 车、自行车（不是动物）

"狗"和"猫"出现在相同上下文 → 它们语义相近 → 向量应该相近
```

### 2. Word2Vec 证明了它（2013）

用神经网络在海量文本上训练，让"出现在相似上下文"的词得到相似向量。
著名验证：

```
king - man + woman ≈ queen    （向量的加减法 = 语义的加减法）
Paris - France + Japan ≈ Tokyo （迁移关系）
```

这证明了：高维向量空间里，**方向和距离都有语义含义**。

### 3. Transformer 让它变成「上下文感知」（2017）

Word2Vec 的问题："bank"（银行/河岸）只有 1 个向量，无法区分词义。

Transformer：
```
"去银行取钱" 里的 "bank" → 向量 A（金融语义）
"坐在河岸边" 里的 "bank" → 向量 B（地理语义）
```
同一个词，不同上下文 → 不同向量。现代 Embedding 模型（BGE/OpenAI）都是 Transformer。

## 1024 维到底编码了什么（可视化理解）

研究者用降维（t-SNE）把 1024 维压成 2 维，能看到自然分簇：

```
              [编程语言区]
           Python •  Java •  TypeScript •
              • Rust    • Go

    [测试工具区]                    [设计工具区]
  Playwright •  • Cypress        Figma •  • Sketch
      • Selenium                    • Adobe XD

              [AI 工具区]
        GPT-4 •  • Claude  • Gemini
              • DeepSeek
```

没有人告诉模型"Playwright 和 Cypress 都是测试工具"——它从数十亿句话里自己学会的。高维向量的每一个方向都在编码某种语义维度（虽然人类看不懂具体是哪个维度）。

## 面试答题模板

> "Embedding 维度代表什么？为什么向量能表达语义？"

```
理论基础是分布假说（1957）：出现在相似上下文的词语义相近。
Word2Vec（2013）验证了可以用神经网络把这种相似性编码成向量。
Transformer（2017）进一步让同一个词在不同上下文有不同向量。

1024 维没有每维的人类可读含义，
但 1024 维加在一起 = 语义空间里的坐标。
维度越高表达力越强，但收益递减：
1024 是效果/成本的行业甜蜜点，够用。
```
