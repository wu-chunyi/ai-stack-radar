# RAG 检索精度评测与优化 · 详解

> 核心问题：Embedding 的精度怎么测？天花板在哪？怎么优化？

---

## 1. 怎么测精度（两个核心指标）

### Recall@K（召回率）

```
前 K 条结果里包含正确答案的比例。
10 个问题搜 top-3，9 个命中 → Recall@3 = 90%
```

### MRR（Mean Reciprocal Rank，平均倒数排名）

```
正确答案排第几位的倒数，取平均。
排第 1 → 1.0 / 排第 2 → 0.5 / 排第 3 → 0.33 / 没搜到 → 0
MRR = 0.9 表示「几乎每次第一条就是对的」
```

### 生产标准

| 等级 | Recall@5 | MRR |
|---|---|---|
| 可上线 | ≥ 85% | ≥ 0.7 |
| 优秀 | ≥ 95% | ≥ 0.85 |

### 我们的实测结果

| 方案 | Recall@3 | MRR |
|---|---|---|
| TF-IDF | 30% | 0.300 |
| **Embedding** | **90%** | **0.900** |

## 2. 精度被什么限制

三层天花板（从底到顶）：

```
┌── Chunking 质量 ──┐   块切得不好，再好的模型也搜不到
│                    │
├── 模型语义能力 ────┤   模型不理解你领域的术语/缩写
│                    │
└── 检索策略 ────────┘   单一检索 vs 混合检索 vs 多路召回
```

---

## 3. 六层优化策略（按性价比排序）

| 优化 | 原理 | 效果 | 难度 |
|---|---|---|---|
| **❶ 改 Chunking** | 块太大→噪音，太小→丢语境。加重叠、按语义切 | +5~15% | ⭐ |
| **❷ Hybrid Search** | BM25关键词 + Embedding语义 混合打分 | +5~10% | ⭐⭐ |
| **❸ Reranking** | 先粗搜20条，用 cross-encoder 精排取 top3 | MRR+0.1~0.2 | ⭐⭐ |
| **❹ 换更好模型** | v3→v4，或换 bge-m3/GTE 等 | 取决于任务 | ⭐ |
| **❺ Fine-tuning** | 用 (问题,正确chunk) 对微调 embedding | 领域内+10~20% | ⭐⭐⭐⭐ |
| **❻ 多路召回** | 标题/内容/关键词 多索引 + 元数据过滤 + LLM融合 | 接近100% | ⭐⭐⭐⭐⭐ |

---

## 4. Hybrid Search 公式

```
final_score = α × normalize(embedding_score) + (1-α) × normalize(bm25_score)
```

α = 0.7 时偏重语义，α = 0.3 时偏重关键词。通常 α = 0.5~0.7 效果最佳。

为什么 Hybrid 几乎总是最优？
- Embedding 搞不定的：精确名称匹配（如搜 "#27" 这种编号）
- BM25 搞不定的：同义词/换种说法
- 两者互补

## 5. 面试答题模板

> "如何评估和优化 RAG 检索质量？"

```
评估：
  1. 建测试集（问题 + 期望命中的 chunk），至少 30~50 条
  2. 算 Recall@K 和 MRR
  3. 分析 bad case：哪些搜不到，是 chunking 问题还是模型问题

优化（按性价比）：
  1. 先调 chunking 策略（块大小、重叠率、切分粒度）
  2. 加 Hybrid Search（BM25 + vector 混合）
  3. 加 Reranking（cross-encoder 精排）
  4. 换更好的 embedding 模型
  5. 极致场景：多路召回 + fine-tune + 人工标注反馈循环
```
